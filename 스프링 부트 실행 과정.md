# 🎨 Spring Boot 시작 과정 시각화 (머릿속 그림)

제공해주신 `IntroApplication.java`의 `main` 메서드가 실행되면, Spring Boot는 **Spring이라는 거대한 컨테이너**를 만들고, 그 안에 웹 애플리케이션 운영에 필요한 부품들을 채워 넣는 과정으로 상상할 수 있습니다.

---

## 1. 시동을 거는 주체: `main()` 메서드

* **상상:** 커다란 **공장 건물** (Java Virtual Machine, JVM) 앞에 서서, 공장 내부의 핵심 시설인 **제어판**을 조작하여 시동을 거는 **기술자**의 모습입니다.
* **코드 역할:** `SpringApplication.run(IntroApplication.class, args);` 이 시동 버튼을 누르는 행위입니다.

## 2. 핵심 시설 구축: IoC 컨테이너 생성

`SpringApplication.run()`이 실행되자마자 Spring Boot가 가장 먼저 하는 일입니다.

* **상상:** 공장 건물 안에 거대한 **투명한 상자** (컨테이너)가 텅 비어 있는 상태로 나타납니다. 이 상자가 바로 **Spring IoC 컨테이너**입니다.
* **역할:** 이 컨테이너는 애플리케이션의 모든 중요한 **부품(Bean)**을 보관하고, 부품들 간의 의존성(연결)을 관리하는 중앙 통제실 역할을 합니다.

## 3. 자동 부품 조립: `@SpringBootApplication`의 마법

`@SpringBootApplication` 애너테이션을 통해 Spring Boot는 필요한 부품이 무엇인지 스스로 판단하고 조립합니다.

* **상상:** 기술자(Spring Boot)가 설계도(클래스패스)를 스캔하며 컨테이너에 부품을 채워 넣습니다.
    * **웹 부품 조립:** `spring-web` 의존성을 선택했다면, **작은 서버 컴퓨터 (내장 Tomcat)** 부품을 컨테이너 안에 넣고 전원을 연결합니다. (→ 웹 요청 처리 준비)
    * **사용자 정의 부품 조립:** 개발자가 `@Service`, `@Controller` 등을 붙인 클래스들은 특정한 기능을 수행하는 **작은 로봇들**처럼 컨테이너 안에 배치되어 서로 연결됩니다.
* **핵심 원리:** **자동 구성(Auto Configuration)**을 통해 개발자가 일일이 설정하지 않아도 되는 환경을 구축합니다.

## 4. 서버 구동 및 대기: HTTP 요청 포트 개방

모든 부품 조립 및 설정이 끝나면, 외부의 요청을 받을 준비를 합니다.

* **상상:** 컨테이너 안에 설치된 **Tomcat 서버 부품**이 번쩍이며 불이 들어오고, 공장 건물 외벽의 **8080번 포트**에 해당하는 창문이 활짝 열립니다.
* **역할:** 서버가 특정 포트(`8080`)에서 **"손님(HTTP 요청)이 오기를 기다리는"** 상태가 됩니다. (로그: `Tomcat started on port 8080`)

## 5. 최종 완료: 실행 완료 로그 출력

* **상상:** 기술자가 제어판의 녹색 불을 확인하고, **"작동 준비 완료!"**라고 외치는 소리(로그 메시지)가 들립니다.

---

### 💡 최종 시각화 요약

**`SpringApplication.run()`**은 "모든 부품이 완벽하게 조립되고, 포트를 개방하여 웹 요청을 처리할 수 있는 **작은 서버 인스턴스**"를 만드는 버튼이라고 상상하시면 됩니다.
