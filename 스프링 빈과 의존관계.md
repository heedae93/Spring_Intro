## 스프링 빈(Spring Bean) 정리

### 1. 정의
* 스프링 IoC(제어의 역전) 컨테이너에 의해 관리되는 자바 객체를 의미한다.
* 개발자가 직접 `new`로 생성하지 않고, 스프링이 생성·조립·관리하는 것이 특징이다.

### 2. 주요 특징
* **싱글톤(Singleton)**: 기본적으로 컨테이너당 하나의 인스턴스만 생성하여 공유한다.
* **의존성 주입(DI)**: 설정된 정보를 바탕으로 객체 간의 의존 관계를 자동으로 연결한다.

### 3. 등록 방법
- **컴포넌트 스캔**: 클래스 위에 `@Component`, `@Service`, `@Controller` 등을 붙여 자동 등록.
```java

// @SpringBootApplication이 붙어 있는 main 메서드가 속해 있는 패키지의 
// 하위 요소들을 전부 스캔한다.
// 해당 범위 밖의 있는 요소들을 별도의 설정을 해주지 않으면 스캔되지 않는다.
package com.spring.intro;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class IntroApplication {

  public static void main(String[] args) {
    SpringApplication.run(IntroApplication.class, args);
  }

} 
```
- **자바 설정 파일**: `@Configuration` 클래스 내에서 `@Bean` 메서드를 통해 수동 등록.

```java
// Bean이 붙은 메서드가 호출되면서 객체 호출하여 등록됨

package com.spring.intro;

import com.spring.intro.repository.MemberRepository;
import com.spring.intro.repository.MemoryMemberRepository;
import com.spring.intro.service.MemberService;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class SpringConfig {
    
    @Bean
    public MemberService memberService() {
        return new MemberService(memberRepository());
    }
    
    @Bean
    public MemberRepository memberRepository() {
        return new MemoryMemberRepository();
    }
    
}


```
